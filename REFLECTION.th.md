# บันทึกสะท้อนการทำงาน

ภาษาไทย | [English](REFLECTION.md)

ส่วนที่ยากที่สุดสำหรับผมคือการทำให้ Prisma 7 และ SQLite เริ่มต้นได้อย่างน่าเชื่อถือบน Windows
Prisma schema engine จะจัดการ schema ได้ก็ต่อเมื่อมีไฟล์ฐานข้อมูลอยู่ก่อน ทำให้ขั้นตอนเริ่มจาก clean checkout แบบปกติล้มเหลว
ผมจึงเพิ่ม wrapper สำหรับตรวจสอบและสร้างไฟล์ SQLite ตาม path ที่กำหนดโดยไม่ล้างฐานข้อมูลเดิม แล้วเรียก Prisma CLI ด้วย URL แบบ absolute
ผมแยกฐานข้อมูลทดสอบและอนุญาตให้ reset ได้เฉพาะ `backend/test.db` เพื่อป้องกันคำสั่งทดสอบลบข้อมูลสำหรับพัฒนา
งานส่วนนี้ทำให้ผมเข้าใจว่า setup script เป็นส่วนหนึ่งของผลิตภัณฑ์ และต้องตรวจ path ทำซ้ำได้ และมี tests เช่นเดียวกับ API endpoint
